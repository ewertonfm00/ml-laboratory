## 3. Decisões tomadas

- **Redis dedup é redundante**: o `ON CONFLICT DO NOTHING` no Postgres garante idempotência; o nó Redis pode falhar sem consequências
- **Fix via API sem mexer no JSON local**: JSON local tem encoding corrompido; atualizar via API é mais seguro e não piora a corrupção existente
- **JSON local desatualizado**: precisa ser sincronizado com a versão em produção (pendência registrada no CONTEXT.md)
