## Decisões técnicas

- **SQL builder no Code node**: `additionalFields.queryParams` é ignorado nesta versão do n8n — construir SQL completo com escaping manual é a única abordagem confiável
- **encerrar-sessao usa `$('Preparar Prompt Claude')`**: evita problema de double-encoding nos nomes dos nós
- **Double-encoding nos nomes**: nós criados com caracteres especiais têm nomes garbled (ex: `Extrair AnÃ¡lise do Claude`) — usar nome garbled exato nas references ou nós ASCII
