## Decisões tomadas

- **Soft delete + Hard delete (Opção C)** para exclusão de parceiros: botão Desativar (toggle `ativo`) + Excluir Definitivamente (modal exige digitar nome exato, validado também no servidor). Hard delete em transação varre 12 tabelas filhas sem `ON DELETE CASCADE`, seta `audit_log.projeto_id = NULL` para preservar histórico, e só então apaga o projeto.
- **Auth admin com cookie HttpOnly** (não Bearer header): cookie `admin_session` HttpOnly+Secure+SameSite=Strict, valor = `ADMIN_API_TOKEN`. Separa segredo (token server-side) de UX (senha humana). `timingSafeEqual` com guard de tamanhos diferentes para constant-time intent.
- **Middleware no edge runtime** usa `===` direto (não `timingSafeEqual`) — risco de timing attack em redirect é desprezível e edge runtime tem caveats com crypto. Helper API mantém constant-time onde realmente importa.
- **`git filter-repo` NÃO foi recomendado**: repo público + senha vazada há 15 dias = qualquer crawler/clone já capturou. Filter-repo é cosmético; rotação é o que de fato resolve.
- **CSV em `projetos.setor`** mantido apenas como info legada — workflow externo lê de `numeros_projeto.setor` (uma row por setor).
